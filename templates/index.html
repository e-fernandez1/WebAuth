What This Site Demonstrates

Each defense below is implemented in this application and exercised when you sign up and log in.

Defense 1

Password Rules

Live client-side validation enforces length, character classes, and rejects common passwords. Same rules re-checked server-side.

Defense 2

bcrypt Hashing

Passwords salted and hashed with bcrypt at cost 12 before reaching storage. A breach exposes hashes, not credentials.

Defense 3

Email 2FA

Cryptographically random 6-digit codes emailed at login. Single-use, 5-minute expiration. Anyone with an inbox can use it — no authenticator app required.

Defense 4

Session Protection

Authenticated routes are gated by signed session cookies. Logout actively invalidates the session.

Threats Addressed

Credential Stuffing

Reused-password attacks fail because 2FA blocks login even when the password is correct.

Database Breach

Stored hashes (not passwords) limit damage. bcrypt's cost factor slows offline brute force to days per password.

Weak Passwords

Live validation enforces strength at account creation, before a weak credential ever enters the system.

Phishing & Replay

Email codes are single-use and expire in 5 minutes, so an intercepted code is useless after one verification or one window.

Session Hijacking

Cryptographically signed session cookies prevent tampering. Logout actively invalidates the session.

Client-Side Bypass

All checks revalidate server-side. Disabling JavaScript does not weaken security.
